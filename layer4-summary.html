2记忆系统工作流程

从对话 → 提取事实 → 保存 → 下次注入，一条完整链路
sequenceDiagram participant MW as 🔄 MemoryMiddleware participant Queue as ⏳ queue.ts participant Msg as 📋 message_processing.ts participant Updater as 📝 updater.ts participant LLM as 🧠 LLM participant Storage as 💾 storage.ts participant File as 📄 memory.json MW->>Queue: 对话完成，加入队列 Queue->>Msg: 过滤消息（去隐藏、去上传） Queue->>Queue: 等 30 秒（debounce） Queue->>Updater: 触发更新 Updater->>Storage: load() 读当前记忆 Storage->>File: 读 memory.json File-->>Storage: 记忆数据 Storage-->>Updater: 当前记忆 Updater->>Updater: 准备提示（过期审查+事实合并） Updater->>LLM: 发送 MEMORY_UPDATE_PROMPT LLM-->>Updater: {"newFacts": [{"content":"偏好 TS"}]} Updater->>Updater: 解析 JSON + 合并到记忆 Updater->>Storage: save(更新后记忆) Storage->>File: 写 memory.json
